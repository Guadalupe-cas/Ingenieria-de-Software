Herramientas CASE

Las herramientas CASE (Ingeniería de software asistida por computadora) son varias aplicaciones de computadora o programas de computadora diseñados para mejorar el equilibrio del desarrollo de software al reducir sus costos de tiempo y dinero. Estas herramientas pueden brindar asistencia en todos los aspectos del ciclo de vida del desarrollo de software, como realizar el diseño del proyecto, contabilidad de costos, usar un diseño dado para implementar automáticamente parte del código, compilación automática, documentación o encontrar errores y otras tareas.

Beneficios

  • Aumentan la productividad
  • Facilitan el trabajo cooperativo
  • Pueden reducir costos de desarrollo

    • Clasificación

  • Interactividad:Interactivas llamadas herramientas CASE. No Interactivas: Herramientas de desarrollo.
  • Fases en que son empleadas: Fron-end:Requerimientos, analisis y diseño.
  • Número de fases del ciclo de vida: Verticales: Una sola fase. Horizontales:Abarca varias faces del ciclo de vida.

    • Fases del proceso de desarrollo

  • Planeación: Antes de iniciar un proyecto de desarrollo de un sistema de información, es necesario realizar determinadas tareas que afectarán decisivamente su éxito. Estas tareas se denominan la parte inicial difusa del proyecto porque no están restringidas por plazos. Algunas tareas en esta etapa incluyen actividades como determinar el alcance del proyecto, realizar estudios de factibilidad, analizar los riesgos relacionados, estimar el costo del proyecto, su plan de tiempo y asignar recursos para las diferentes fases del proyecto. proyecto.
  • Analisis: Por supuesto, debe averiguar qué debe hacer el software. Por lo tanto, la fase de análisis del ciclo de vida del software corresponde al proceso de intentar descubrir qué se necesita realmente y comprender completamente los requisitos del sistema (las características que debe tener el sistema).
  • Diseño: En esta etapa se estudiará la posible implementación del software a construir y se determinará su estructura general. El diseño es una etapa compleja y el proceso debe completarse de forma iterativa. Es posible que la solución inicial no sea la más adecuada, por lo que en este caso se debe refinar. Sin embargo, existe un catálogo de patrones de diseño muy útil para recoger errores cometidos por otros para no caer en la misma trampa.
  • Implementación: En esta etapa, debe elegir las herramientas adecuadas, un entorno de desarrollo que facilite el trabajo y el lenguaje de programación adecuado para el tipo de software que desea construir. Esta elección dependerá de las decisiones de diseño que se tomen y del entorno en el que se ejecuta el software.
  • Pruebas: La fase de prueba del ciclo de vida del software tiene como objetivo detectar errores en fases anteriores para su corrección.
